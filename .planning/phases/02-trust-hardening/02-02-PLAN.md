<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| worker -> main (attachment) | Refused + healed events cross the process boundary as JSON on the `selfmend-heal` attachment. |
| live failure -> heal decision | The proxy is where the real TimeoutError is observed and where the no-false-green guarantee is enforced at runtime. |
| attachment -> reporter | The reporter parses untrusted attachment bytes in the main process. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Tampering | refused-event attach in locator-proxy.ts | mitigate | Attach the refused event THEN unconditionally re-throw the original error (D-06); the attach never gates the throw, so observability cannot produce a false green (Pitfall 2). Integration test asserts the action rejects. |
| T-02-05 | Denial of Service | reporter `parseEvent` on attachment bytes | mitigate | Validate every field per branch and skip malformed entries (existing T-05-02 pattern); never throw in `onTestEnd`. Missing `kind` falls back to healed rather than crashing (back-compat). |
| T-02-06 | Information Disclosure | RefusedEvent payload | mitigate | Carries only originalSelector, reason, and a numeric bestScore — derived signals only, never raw DOM content (T-05-03 consistent). |
| T-02-07 | Spoofing/Tampering | back-compat of the wire contract | mitigate | Missing-`kind` attachments parse as healed; old Phase 1 healed attachments remain valid (Pitfall 4), so the contract widens without breaking in-flight payloads. |
| T-02-SC | Tampering | npm/pip/cargo installs | mitigate | No net-new packages this phase (RESEARCH Package Legitimacy Audit: N/A); no install task, so no slopcheck checkpoint required. |
</threat_model>
