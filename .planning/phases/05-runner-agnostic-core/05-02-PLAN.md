<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller -> wrapPage opts (raw mode) | The raw-mode adopter supplies store/config/onHeal/scope; the adapter (fixture) supplies a trusted emit, but the public surface must tolerate untrusted ones. |
| heal path -> emit/onHeal | Caller onHeal runs inside the heal path; it must never affect the run. |
| worker -> main (attachment) | The fixture adapter's emit crosses the worker->main boundary via testInfo.attach, carrying derived audit fields only (no DOM content). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | a throwing `onHeal` masking/breaking the run | mitigate | onHeal is fire-and-forget with throws swallowed (Plan 01 emit); control-tested here (Task 2 fail-safe case): a throwing onHeal still heals green and never breaks the run (D-07). |
| T-05-02 | Denial of Service | a throwing `scope()` crashing the page wrap | mitigate | the scope-lifetime controller (Plan 01) falls back to the coarse default on a throwing scope(); control-tested here (Task 2 fail-safe): a throwing scope() does not crash the wrap and the run proceeds (D-04, D-11). |
| T-05-04 | Spoofing | a wrong/missing identity key producing a false green in raw mode | mitigate | control-tested here (Task 2 wrong-scope + absent-element): a mismatched/absent key yields a MISSED heal and the action fails normally — never a wrong heal, never a false green (D-11, the cross-cutting hard rule). |
| T-05-05 | Information Disclosure | the fixture adapter's emit leaking DOM content across worker->main | mitigate | the adapter reuses the existing attachHealEvent/attachRefusedEvent which carry derived audit fields only (selectors + score + reason), never raw DOM (existing T-02-06); byte-identical attachments preserve this (D-09). |
| T-05-SC | Tampering | npm/pip/cargo installs | n/a | No new runtime dependencies (refactor + test only). No install tasks. |
</threat_model>
