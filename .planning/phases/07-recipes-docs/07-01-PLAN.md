<objective>
Build the compilable-examples spine that makes every README recipe runnable as written (ROADMAP Phase 7 success criterion 3). Three TypeScript example files (Cucumber, Mocha/Jest, plain script) become the SINGLE SOURCE OF TRUTH for the recipe code: each imports the package's published API and exercises the real wrapPage / scope() / loadBaseline / saveBaseline / renderHealSummary surface. A dedicated tsconfig plus an npm smoke-check type-check them against the BUILT package, and a CI step runs that smoke-check so a recipe can never silently rot.

This plan deliberately contains NO prose recipes and NO README edits. It produces only the machine-checked example files and the gate. Plan 07-02 writes the README around these files and a sync check binds the two.

Purpose: Criterion 3 is the load-bearing engineering task. If the examples do not compile against the real types, the recipes are wrong on day one.
Output: examples/ dir (3 recipes + 1 type-shim), tsconfig.examples.json, scripts/check-examples.mjs, a check:examples npm script, and a CI step.
</objective>

    WrapPageOptions = {
      store: BaselineStore;            // required
