<objective>
Expose the standalone persistence building blocks a raw-framework user needs: load a baseline from a literal file path, save it refresh-and-add only, and merge per-worker baselines deterministically, all decoupled from the Playwright reporter and shard machinery. Every piece is a thin wrapper over already-tested internals (parseBaseline, serialize, atomicWrite, refresh, mergeShards); this plan adds wrappers and the public name, not new logic.

This is the persistence vertical slice. After it, a developer driving a raw Playwright Page can persist a baseline to a file of their choosing and heal off it on a later run with no reporter, no shards, no @playwright/test runner.

Purpose: Deliver STORE-01, STORE-02, STORE-03 by re-exposing the fs adapter and pure merge layer, and resolve the loadBaseline name collision so the public path-based name is free.
Output: Public loadBaseline(path) / saveBaseline(path, store) / mergeBaselines(...stores) exported from src/index.ts, the internal rootDir-based loader renamed to loadCommittedBaseline, an additive survival-invariant unit test, a deterministic merge unit test, and a Playwright e2e proving save-then-reload-then-heal off the file alone.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller filesystem path -> saveBaseline/loadBaseline | The consumer supplies a literal baseline file path; selfmend reads and writes it |
| on-disk baseline file -> parseBaseline | An untrusted (possibly hand-edited or stale-version) JSON file is parsed back into a store |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | loadBaseline(path) reading a malformed / hand-edited baseline file | mitigate | Route the parse through parseBaseline (zod strict, version-gated, fail-soft to EMPTY, never throws); no hand JSON.parse field-trusting |
| T-06-02 | Tampering | loadBaseline reading a foreign / old STORE_FORMAT_VERSION file | mitigate | parseBaseline's z.literal(STORE_FORMAT_VERSION) rejects to EMPTY (ignore-and-recapture), so a stale file never half-loads |
| T-06-03 | Tampering | saveBaseline path traversal via caller-supplied path | accept | The PUBLIC saveBaseline(path) trusts the consumer's own path by design (their project, their choice); the internal env-override path keeps its existing storeRoot containment clamp. Documented distinction, no new attack surface |
| T-06-04 | Information disclosure | persisted file leaking raw DOM | mitigate | The serializer emits derived signals only (schema.ts strict object); unchanged here, no raw DOM ever written |
| T-06-05 | Tampering | a half-written baseline file on an interrupted save | mitigate | saveBaseline reuses atomicWrite (temp-file + rename with the Windows EPERM/EBUSY/EACCES retry loop); a reader never sees a partial target |
| T-06-SC | Tampering | npm/pip/cargo installs | accept | This phase installs NO packages (source-only addition; runtime deps stay zod + picocolors). Package Legitimacy Gate N/A |
</threat_model>
