<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| test attachments → reporter (main process) | Heal events deserialized in the main process |
| public entry → consumer test suite | The re-exported test is the plugin's trust surface |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Integrity | reporter accidentally healing | mitigate | Reporter is summary-only (D-05); reads attachments, has no DOM/page access, cannot rebind |
| T-05-02 | Tampering | malformed selfmend-heal attachment | mitigate | Parse attachments defensively; skip/ignore malformed entries rather than crash the run |
| T-05-03 | Information disclosure | heal summary leaking page text | mitigate | Summary prints original selector + healed target + score only — derived signals, not raw DOM content |
| T-05-04 | Integrity | assertions routed through heal | mitigate | install.spec asserts expect(...) does not heal; action-method partition (plan 04) keeps assertions sacred |
</threat_model>
